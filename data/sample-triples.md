## 使用方法

### 方法1: 通过前端界面导入

1. 启动系统后，点击顶部"导入数据"按钮
2. 选择"Markdown表格"标签
3. 复制上面的表格内容（包括表头）
4. 粘贴到输入框中
5. 点击"导入"按钮

### 方法2: 通过API导入

```bash
curl -X POST http://localhost:3001/api/v1/import/markdown \
  -H "Content-Type: application/json" \
  -d '{
    "content": "表格内容...",
    "type": "triples"
  }'
```

## 实体ID命名规范

- `PROJ_xxx`: 车型项目
- `SSTS_xxx`: 系统需求
- `SYS_xxx`: 系统架构
- `SWR_xxx`: 软件需求
- `PF_xxx`: 感知融合模块
- `OD_/PD_/Lane_xxx`: 模型版本
- `Daily_/MRD_xxx`: 发布包
- `TC_xxx`: 测试用例
- `ISSUE_xxx`: 测试问题

## 关系类型

- **管理**: 车型项目管理系统需求
- **拆解为**: 需求/架构分解为子项
- **指导**: 软件需求指导开发模块
- **产出**: 开发模块产出模型/代码
- **集成于**: 模型集成到发布包
- **验证于**: 需求被测试用例验证
- **发现**: 测试发现问题
- **关联至**: 问题追溯到需求
